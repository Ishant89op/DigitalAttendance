# Smart Digital Attendance Management System

## Project Overview

The Smart Digital Attendance Management System is a biometric-based attendance platform designed to automate attendance recording in academic institutions using facial recognition.

The system replaces traditional attendance registers with a secure digital solution that integrates computer vision, backend APIs, and a web dashboard.

It provides automated attendance marking, subject-wise analytics, faculty monitoring tools, and administrative control for managing institutional data.

The architecture is designed to be modular so that it can adapt to different academic structures where semesters, subjects, and course assignments vary across institutions.

---

# Core System Modules

## Face Recognition Engine

This module detects and recognizes student faces using a webcam.

Process:

1. Camera captures frames continuously.
2. Face detection identifies faces in the frame.
3. Face encoding converts detected faces into numerical embeddings.
4. Encodings are compared with stored biometric templates.
5. If a match is detected, the student identity is verified.

Face recognition uses deep learning based embeddings generated by the face_recognition library.

---

## Biometric Template Storage

Each student has a unique biometric template stored in the database.

Face encodings are 128-dimensional vectors representing facial features.

Example encoding structure:

[0.23, -0.12, 0.44, 0.91, ... 128 values]

For security purposes:

- Only biometric templates are stored
- Raw images are not stored

---

## Liveness Detection (Future Feature)

To prevent proxy attendance using photos, the system will include liveness verification.

Possible methods:
- Blink detection
- Head movement detection
- Temporal face movement validation

Attendance will only be recorded after successful liveness verification.

---

## Attendance Recording System

Once a student is recognized, attendance is automatically recorded.

Attendance logic:

1. Student face recognized
2. System verifies active class session
3. Duplicate attendance is checked
4. Attendance entry recorded with timestamp

Example attendance entry:

Student ID: 101  
Subject ID: CS401  
Status: Present  
Timestamp: 2026-02-15 09:05:22

# Semester Based Subject Mapping

Institutions may have different subjects per semester.

To support this, the system links students to subjects automatically through semester mapping.

Example:

Semester 4

CS401 – Computer Networks  
CS402 – Operating Systems  
CS403 – Database Systems

Student registered in Semester 4 will automatically be associated with these subjects.

This allows the recognition system to mark attendance under the correct subject without manual configuration.

# Student Registration Workflow

Student data is first created by the administrator.

Admin creates student record:

Student ID: 101  
Name: Rahul  
Semester: 4  
Department: CSE  

After the record exists, the student performs biometric enrollment.

Enrollment process:

1. Student enters Student ID
2. System verifies student record
3. Face samples are captured
4. Biometric template stored in database

The face encoding is linked directly to the student profile.

# Recognition Optimization

To improve performance and reliability the system includes:

Frame Downscaling

Frames are resized to 25% resolution to improve speed.

Blur Detection

Blurred frames are rejected using Laplacian variance.

Recognition Buffer

Faces must appear consistently across multiple frames before recognition is confirmed.

Cooldown Timer

Prevents repeated attendance marking within short intervals.
